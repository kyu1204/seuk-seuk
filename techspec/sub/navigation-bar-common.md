# 공통 네비게이션 바 구현 기술 명세서

## 개요
각 페이지마다 개별적으로 구현된 헤더/네비게이션을 공통 컴포넌트로 통합하여 일관된 사용자 경험 제공

## 현재 상황 분석

### 기존 구현 현황
1. **홈페이지 (app/HomePage.tsx)**
   - 자체 header 컴포넌트 (101-122행)
   - 로그인 버튼 포함
   - 스크롤 시 배경색 변경 효과

2. **대시보드 (app/dashboard/DashboardPage.tsx)**  
   - 별도 헤더 영역 (221-278행)
   - 사용자 아바타 + 드롭다운 메뉴
   - 언어 선택기 + 로그아웃 기능

3. **로그인/회원가입 페이지**
   - 각각 우상단에 별도 네비게이션 버튼들
   - 홈으로 돌아가기, 로그인/회원가입 전환 버튼

### 문제점
- 코드 중복
- 일관성 없는 디자인
- 유지보수 어려움
- 사용자 상태에 따른 네비게이션 로직 분산

## 설계 방안

### 1. 공통 네비게이션 컴포넌트 구조
```tsx
interface NavigationBarProps {
  // 추가 props가 필요한 경우를 대비
}

// 로그인 상태에 따른 조건부 렌더링
// - 로그인 전: 로고 + 테마토글 + 언어선택 + 로그인버튼  
// - 로그인 후: 로고 + 테마토글 + 언어선택 + 사용자아바타
```

### 2. 사용자 상태 관리
- `useUser` 훅 활용하여 현재 사용자 정보 획득
- 클라이언트 사이드 렌더링 고려
- 로딩 상태 처리

### 3. 배치 위치
- `app/layout.tsx`의 Provider 내부에 배치
- 모든 페이지에서 공통으로 표시
- sticky 포지션으로 상단 고정
